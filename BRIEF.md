Designing a talk and set of slides around the topic of:
Effect Systems make Agentic Development Better (To be made more punchy)

15 minute lightning talk

language agnostic, but with examples in haskell+effectful (my preference) - but can talk about other languages and approaches for other code bases (pros / cons)

benefits:
- defining a strict architecture (e.g. a backend API) using well defined layers: API Handlers -> Domain Layer -> Effects -> Interpreters -> [DB, Logging, External APIs, etc] means that Agents writing code and adding new features will be provided strict guardrails and compile-time checking of architecture pattern
- reviewing of code generated by agents much easier when effects are reflected in the type system - especially as they taint - you can't hide effects - you are forced to define them properly
- agents can leverage the compiler to guide them towards correct implementations (even with small context windows, the architecture can be described very succinctly by the types at each layer)
